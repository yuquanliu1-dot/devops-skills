# 创建新子技能工作流

当用户要求创建一个新的 PPT 子技能（sub-skill）来应对某种特定场景（例如：年终总结、新产品发布、入职培训等）时，在实际运行脚手架脚本之前，你**必须**先与用户进行一轮**配置需求访谈**。

请向用户发问，明确以下定制化细节，以便你稍后将这些设定沉淀到该子技能的规范中：

1. **面向的听众（Audience）**：这个场景的典型听众是谁？（例如：公司高管、外部客户、低年级学生？这决定了默认的文案语调、用词深浅和图文比例分布。）
2. **整体配色与视觉（Color Scheme / Theme）**：期待什么样的视觉基调？（活泼多彩、清新自然、科技感还是严谨商务？这决定了后续推荐的 template 和 archetype 的选用。）
3. **讲解节奏（Pacing / Duration）**：常规情况下的讲解时长约束是多少？PPT 重点应该放在哪些核心环节？（例如：是强调成果展示，还是侧重于痛点剖析？）
4. **配图与排版（Visuals / Images）**：是否有比较明确的配图要求、特有组件需求（例如非常依赖表格对比、或者需要大量自动生成的配图）？

## 执行步骤

1. **发起访谈**：第一步，不要运行任何代码，先主动询问上述 4 个配置需求。
2. **运行脚本创建脚手架**：在用户确认具体设定后，调用自动生成脚本：
   ```bash
   node scripts/create_ppt_skill.js <slug> "<Display Name>"
   ```
3. **微调落地（关键）**：脚本执行完成后，根据用户刚才回答的内容，你必须**手动修改**以下这几个刚刚生成的文件，把约定的配置写死进去：
   - `subskills/<slug>/SKILL.md`：把用户设定的默认受众要求、建议时长、图文要求等补充在“适用场景”或“内容提醒”相关片段中。
   - `subskills/<slug>/references/outline.md`：按用户要求的重点环节和节奏设定，填充或重写细化大纲结构（不要保留默认的脚手架大纲）。
   - `assets/archetypes/<slug>.json`：让默认模版符合用户期望的时间分配，调整或增加幻灯片页面数量和结构，并确保 `meta.theme` 符合约定的配色风格。
