# 个人介绍

这个子 skill 用于个人介绍、嘉宾介绍、求职陈述和作品集首页。重点不是把简历逐条贴进 PPT，而是用 4 到 6 页讲清楚“我是谁、为什么值得被记住、我做过什么、我想如何被合作”。

## 何时使用

- 用户要做个人介绍、自我介绍、嘉宾介绍
- 用户要做作品集首页、个人陈述、求职展示
- 用户希望突出人物标签、经历、项目和风格

## 默认选择

- archetype：`personal-intro`
- template：优先 `profile-editorial`，也可选 `launch-stage`
- 推荐页数：`4-6 页`

## 推荐起稿命令

```bash
node scripts/init_project.js personal-profile "个人介绍" --template profile-editorial --archetype personal-intro
```

## 内容重点

- 第一页要让观众快速记住“你是谁”
- 个人标签要控制在 3 个以内，写成可记忆表达
- 经历页不要写成简历全文，优先保留转折和亮点
- 代表项目页优先一页只讲一个代表案例
- 结尾页要给一句能被记住的话，最好带合作方式或价值主张

## 输入时优先补齐的信息

- 你的身份定位、目标听众、希望别人记住的 3 件事
- 代表经历或关键节点
- 1 到 2 个最值得讲的项目
- 个人照片、作品截图、Logo 或个人品牌素材

## 组件与文案规则

- 优先使用 `cover`、`title-bullets`、`timeline`、`two-column`、`quote`
- 一页最多一个核心信息块，不要同时塞经历、项目和技能
- 文案偏口语化、可信、克制，不要写成夸张广告词
- 项目页优先写“做了什么 + 为什么重要 + 带来了什么结果”
