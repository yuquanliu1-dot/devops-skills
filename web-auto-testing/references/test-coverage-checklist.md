# 测试覆盖率检查清单

本文档用于自检测试计划的覆盖率，确保不遗漏关键测试点。

## 使用说明

在完成测试计划后，逐项检查以下清单，确保所有项目都已覆盖。

---

## 需求覆盖率检查

### 需求-测试点映射检查

- [ ] 每个需求都有对应的测试点
- [ ] 核心需求（P0）有详细测试
- [ ] 需求的所有约束条件都有测试
- [ ] 需求的所有边界条件都有测试

### 需求覆盖率计算

```
需求覆盖率 = 已覆盖需求数 / 需求总数 × 100%
```

- 需求总数：_____
- 已覆盖需求数：_____
- 需求覆盖率：_____%

### 目标标准

| 测试类型 | 需求覆盖率目标 |
|---------|---------------|
| Smoke | ≥ 80% (核心需求) |
| 回归 | ≥ 90% (变更相关需求) |
| 完整 | 100% (所有需求) |

---

- [ ] 核心业务流程已覆盖
- [ ] 高频使用功能已覆盖

### 识别的交互元素

- [ ] 数据量边界：空数据、单条、大数据量

### 异常数据检查

- [ ] 异常场景有对应的异常数据
- [ ] 空值测试数据
- [ ] 超长输入测试数据
- [ ] 特殊字符测试数据

### 数据来源确认

- [ ] 认证数据来源已确认（用户提供）
- [ ] 业务数据来源已确认（用户提供）
- [ ] 基础数据可智能生成
- [ ] 边界数据可智能生成
- [ ] 异常数据可智能生成
