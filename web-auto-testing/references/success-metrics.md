# 测试成功指标

本文档定义测试通过的标准和部署决策依据。

---

## 🎯 分级标准

| 测试类型 | 通过率 | 不稳定率 | 说明 |
|---------|--------|----------|------|
| **Smoke** | 100% | 0% | 核心功能必须全部通过 |
| **回归测试** | ≥ 95% | < 5% | 允许少量非关键失败 |
| **完整测试** | ≥ 90% | < 10% | 覆盖面广，容许率略高 |

---

## 📊 关键指标

### 必须指标（阻止部署）

- ✅ 关键旅程通过率 = 100%（登录、支付、核心流程）
- ✅ 无新增失败测试（相比基线）
- ✅ 无关键 Bug（P0、P1 级别）

### 质量指标（应达到）

- ✅ 总体通过率 ≥ 95%
- ✅ 不稳定率 < 5%
- ✅ 测试产物完整（截图、视频、追踪已上传）
- ✅ HTML 报告可访问

### 性能指标（建议）

- ✅ 测试持续时间 < 15 分钟（并行执行）
- ✅ 测试套件大小合理（< 500 个测试用例）

---

## 🚦 部署决策

### 可以部署

```
✅ 关键旅程通过率 = 100%
✅ 总体通过率 ≥ 95%
✅ 无新增失败测试
✅ 无关键 Bug
```

### 需要评估

```
⚠️ 关键旅程通过率 < 100%
⚠️ 总体通过率 90-95%
⚠️ 有新增失败但非关键功能
```

**行动**：评估失败影响，决定是否推迟部署

### 禁止部署

```
❌ 关键旅程失败
❌ 总体通过率 < 90%
❌ 新增关键 Bug
```

---

## 📈 测试稳定性监控

### 不稳定测试识别

**定义**：同一测试在 3 次运行中至少失败 1 次，且不是同一原因

**处理**：
1. 标记为不稳定
2. 修复或从基线中移除
3. 不计入部署决策

### 稳定性提升策略

| 问题类型 | 解决方案 |
|---------|----------|
| 选择器不稳定 | 使用更稳定的选择器（getByRole） |
| 网络超时 | 增加超时时间，添加重试逻辑 |
| 并发冲突 | 使用独立数据，加锁机制 |
| 依赖顺序 | 确保测试间无依赖 |
